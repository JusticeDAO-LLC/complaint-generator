Intake Questions

Capture the facts once, then reuse them everywhere else in the workflow.

Start with the story the complainant can tell clearly today

Use plain language. If dates or exact wording are uncertain, capture the best current version and come back with corroboration in the Evidence stage.

Evidence Intake

Collect both testimony and documents, and attach each item to a claim element so support gaps are obvious.

What counts as helpful support

Prefer concrete sources: emails, messages, witness accounts, meeting notes, timeline records, HR filings, performance reviews, and anything that helps anchor protected activity, employer knowledge, causation, or damages.

Support Review

See which claim elements are already supported and which ones still need more evidence before the complaint is finalized.

Draft and Edit

Generate the complaint from the workspace state, then make final edits without leaving the same flow.

Draft when the case theory is coherent

A usable draft needs a stable plaintiff identity, an opposing party, protected activity, adverse action, timing, and harm. If one of those pieces is weak, use Review and Evidence first.

CLI and MCP Tools

The website is using the JavaScript MCP SDK below. The same service is also available through a Python CLI and a lightweight stdio MCP tool server.

Python package imports

from complaint_generator import ComplaintWorkspaceService, create_review_surface_app
from complaint_generator.mcp import handle_jsonrpc_message

service = ComplaintWorkspaceService()
session = service.get_session("did:key:example")

Python CLI

complaint-workspace session --user-id did:key:example
complaint-generator-workspace session --user-id did:key:example
complaint-workspace answer --user-id did:key:example --question-id protected_activity --answer-text "Reported discrimination to HR"
complaint-generator-workspace generate --user-id did:key:example --requested-relief "Back pay|Injunctive relief"

MCP stdio server

complaint-generator-mcp
complaint-mcp-server

JavaScript MCP SDK

const client = new window.ComplaintMcpSdk.ComplaintMcpClient();
const bootstrap = await client.bootstrapWorkspace();
await client.callTool('complaint.review_case', { user_id: bootstrap.did });
